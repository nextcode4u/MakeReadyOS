# UX Priorities

## Operational Principle

The source board is used as a dense working surface. MakeReadyOS should optimize for rapid scanning, status editing, and schedule-risk recognition rather than generic task-management presentation.

## Highest-Priority Behaviors

- Show many usable rows and columns on a laptop screen.
- Keep key identity and header context visible while scrolling.
- Make colored status values readable at a glance.
- Keep inline edits fast for repeated daily updates.
- Support property and workflow-group filtering without hiding the working table.
- Make date-based scheduling visible through dedicated calendar views.

## View Priorities

| View | Primary Job |
| --- | --- |
| Table | Dense daily editing and oversight across trades and dates |
| Calendar | Detect workload and move-in scheduling pressure |
| Kanban | Move work through selected lifecycle/status categories |

## Interaction Guidance

- Compact Mode should remain available for operators reviewing large boards.
- Filters and search belong close to view navigation and should consume little vertical space.
- Destructive or lifecycle-impacting actions should remain explicit and auditable.
- Mobile layouts must retain readable status pills and adequate touch targets, even if less dense than desktop.

## Accessibility Baseline

- Do not rely only on color to communicate state.
- Keep keyboard focus visible.
- Preserve text labels for all editable statuses and toolbar actions.
- Maintain readable contrast in dense dark-mode tables and calendars.
