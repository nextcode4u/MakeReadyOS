# Support

MakeReadyOS is an open-source, self-hosted project. Public support is best-effort unless a separate support agreement exists.

## Before Asking For Help

1. Read [README.md](README.md).
2. Run `./doctor.sh`.
3. Check [docs/BUILD_AND_TEST.md](docs/BUILD_AND_TEST.md).
4. Check [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md).
5. Review recent logs under `logs/`.

## Good Support Requests Include

- MakeReadyOS commit or release tag
- deployment type: Docker, Raspberry Pi, VM, VPS, local dev
- browser and operating system
- command that failed
- relevant log file excerpt
- whether demo data or production data is involved

Do not post secrets, `.env` contents, resident data, staff data, vendor credentials, backup files, or uploaded attachments.

## Where To Go

- Bugs: open a GitHub issue using the bug report template.
- Feature ideas: open a GitHub issue using the feature request template.
- Security issues: follow [SECURITY.md](SECURITY.md) and report privately.

## Commercial Or Production Use

The project can be self-hosted, but operators should validate backups, restores, roles, permissions, and deployment security before using it for real property operations.
