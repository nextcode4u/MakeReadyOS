# Open Source Guide

## Documentation Expectations

This project should stay understandable to:

- property operations end users
- contributors
- future maintainers

## Documentation Rules

- keep docs concrete and operational
- avoid stale feature claims
- update `reference/project/PROJECT_REFERENCE.md` when project direction changes materially
- update `docs/ROADMAP.md` when scope priorities change
- update `README.md` when setup or deployment steps change

## Release Readiness

The repository now includes a concise root README, contributor guide, security policy, support guide, issue templates, and committed screenshot examples under `docs/screenshots/`.

Before public release, still finalize:

- the project license in `LICENSE`
- an optional code of conduct
- release tags and changelog/release notes
- a restore rehearsal for a disposable deployment
